제안 포인트

1. 인쇄 규격 정합성을 1순위로 하는 설계

  • 인쇄 사양(사이즈·DPI·블리드·CMYK·ICC)을 단일 PrintSpec 객체로 추상화하여, 사이즈 변경(예: 8.5×8.5 → 다른 규격)이나 POD 업체 변경 시 코드 수정 없이 설정값만 교체 가능
  • 최종 PDF는 PDF/X-1a 호환 메타데이터를 부여하여 인쇄소가 별도 변환 없이 입고 가능한 상태로 출력

2. 색관리(Color Management) 정확도

  • Pillow + LittleCMS 기반 sRGB→CMYK 변환 파이프라인 구성, ICC 프로파일(USWebCoatedSWOP)을 명시적으로 적용하여 모니터-인쇄 간 색차 최소화
  • 이미지별 변환 후 픽셀 비교 테스트로 색공간 변환 회귀 방지

3. Vercel Serverless 호환 모듈 구조

  • 모듈은 외부 바이너리 의존성(예: ImageMagick) 없이 순수 Python 라이브러리(Pillow, fpdf2/ReportLab, qrcode, reportlab.graphics) 위주로 구성하여, Vercel·AWS Lambda 같은 서버리스 환경에서 단일 wheel 패키지로 배포 가능
  • 호출 인터페이스는 build_book(images, story, cover, spec) → bytes 형태의 순수 함수로, 콜드스타트 영향을 최소화하고 메모리 풋프린트를 측정·문서화하여 제공

4. 테스트·문서·샘플의 풀 패키지 납품

  • pytest 기반 단위 테스트(레이아웃·CMYK 변환·QR·표지 조립·서버리스 호출), 샘플 입출력(이미지 20장 + 스토리 JSON + 결과 PDF) 동봉
  • README는 설치·CLI·Python API·Vercel 함수 예제·트러블슈팅 섹션으로 구성하여 발주처 인계 후 추가 학습 비용 최소화

5. 정부지원사업 운영 적합성

  • 견적서 발행, 보증보험 발급, 단계별 산출물 인계, 회의록·검수서 양식 표준화로 정부지원사업 정산 절차에 바로 대응 가능
  • 선금-중도금-잔금 3단계 지급 일정에 맞춘 마일스톤 검수 체계 제안

PRD (Product Requirements Document)

POD 커스텀 아동 동화책 PDF 자동 조립 모듈의 기능 범위와 요구사항을 정의합니다.

1. 프로젝트 배경 및 목적

POD(Print-on-Demand) 커스텀 아동 동화책 서비스에서, 사용자별로 생성된 이미지·스토리·표지 정보를 입력받아 인쇄소에 입고 가능한 8.5×8.5인치 하드커버용 인쇄 규격 PDF를 자동 생성하는 Python 모듈을 개발합니다. 본 모듈은 발주처의 전체 서비스 아키텍처에서 "PDF 조립" 단계의 핵심 컴포넌트로, Vercel Serverless 환경에서 호출되어 최종 PDF를 출력하고 선택적으로 POD 업체 API에 주문을 발행합니다.

정부지원사업 과제로 수행되며, 견적서 발행 및 보증보험 발급 등 정산 절차를 준수합니다.

2. 사용자 정의

사용자 유형설명주요 기능
발주처 백엔드전체 서비스(주문·콘텐츠 생성)에서 본 모듈을 호출하는 상위 시스템모듈 호출, 입력 검증, 출력 PDF 수신, POD 주문 발행 트리거
발주처 운영자주문 검수·재발행·CS 처리 담당샘플 PDF 미리보기, 인쇄 규격 검증 결과 확인

3. 기능 요구사항

3.1 입력 / 출력

  • [필수] 입력: 이미지 20장(PNG, 4096×4096px, sRGB), 스토리 텍스트 JSON(페이지별 본문·폰트·위치·색상), 표지 정보(제목·아이 이름·부제), 인쇄 규격 옵션
  • [필수] 출력: 인쇄용 PDF 1개(표지 + 내지 20페이지, PDF/X 호환 메타데이터 포함)
  • [필수] 입력 검증: Pydantic 기반 스키마 검증, 이미지 해상도·DPI·색공간 사전 점검

3.2 표지 생성

  • [필수] 앞 표지: 표지 이미지 + 제목 + 아이 이름 + 부제 자동 배치
  • [필수] 뒤 표지: 로고, QR 코드(주문 고유 ID 기반), 저작권 텍스트
  • [필수] OTF / TTF 커스텀 폰트 임베딩

3.3 내지 레이아웃

  • [필수] 페이지별 풀블리드 / 마진 모드 분기
  • [필수] 텍스트 위치·색상·반투명 박스 옵션 (페이지별 JSON 지정)
  • [선택] 페이지 번호 표시 토글

3.4 인쇄 규격 처리

  • [필수] 8.5 × 8.5인치 (사이즈 변동 가능 — 설정값 교체로 대응)
  • [필수] 300 DPI 해상도 보장, 미달 이미지 경고
  • [필수] CMYK 색공간 (sRGB→CMYK 자동 변환 포함)
  • [필수] 블리드 0.125인치 (사방), 트림박스·안전영역 메타데이터 출력
  • [필수] ICC 프로파일 USWebCoatedSWOP 적용

3.5 QR 코드

  • [필수] 주문 고유 ID 기반 QR 코드 동적 생성
  • [필수] 뒤 표지 지정 위치에 자동 삽입

3.6 POD API 연동 (선택)

  • [선택] Printful 또는 Blurb API로 인쇄 주문 자동 발행
  • [선택] 주문 상태 추적 webhook 수신 샘플 (서명 검증 포함)
  • [선택] POD 업체 변경 시 어댑터 교체로 대응 가능한 구조

3.7 서버리스 호환

  • [필수] Vercel Serverless 함수에서 호출 가능 (단일 wheel 패키지)
  • [필수] 외부 바이너리(ImageMagick 등) 의존 없는 순수 Python 라이브러리 우선 사용
  • [필수] 콜드스타트·메모리 풋프린트·실행 시간 측정 결과 문서화
